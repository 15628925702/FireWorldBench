# FireWorldBench v1 项目章程

状态：`ACTIVE`  
版本：`0.1.0`  
日期：`2026-07-14`

## 1. 研究目标

构建一个可复现、可审计、可扩展的火灾物理世界理解 benchmark，用统一协议评估模型在以下三类能力上的表现：

1. T1 火灾预警：弱信号阶段的风险判断、异常归因和主动信息选择。
2. T2 火灾状态识别：火源/阶段/风险状态、主导机理和解释物理一致性。
3. T3 火灾演化推理：未来趋势、单变量反事实和中间状态转移链。

数据集不是任务本身。v1 先定义统一的能力 ontology 和样本/评分协议，再把异构数据源按其可提供的观测、真值和验证角色映射到 T1/T2/T3。简单问答只是一种输入输出形式，不能替代物理状态、机制、证据、时序和反事实能力定义。

## 2. 核心研究问题

- RQ1：现有 LLM/multimodal agent 能否从有限、多模态、带噪观测中恢复隐含火灾状态，而非只做表面检测？
- RQ2：显式物理轨迹、检索、绘图或工具使用能否稳定降低物理违规并提高 OOD 泛化？
- RQ3：模型是否对“只改变一个干预变量”的反事实产生方向正确、证据一致的响应？
- RQ4：传统时序模型、视觉模型和 LLM/agent 在预警、识别、演化三类任务上的能力边界分别是什么？

## 3. v1 范围

- 主数据：Immersed-Tunnel-CFD 抽样；纸面可发表版本必须补齐经许可的数据版本与来源快照。
- 真实实验验证：PolyUFire sidewall vent 与 FDS experimental validation 子集。
- 受控演化：FDS + FD-Gen；在可复现运行与成本门禁通过后进入 paper-ready 主实验。
- 视觉辅助：优先 D-Fire；DetectiumFire 仅在 non-fire/许可/样本完整性门禁通过后加入。
- 任务：T1-A/B/C、T2-A/B/C、T3-A/B/C，MVP 可按门禁分批交付但不可改变最终定义。
- 组织方式：能力轴固定为 T1/T2/T3；数据轴区分仿真、受控实验、传感器时序、视觉和多模态组合，两条轴正交组合而非一数据集一任务。
- 领域边界：v1 只覆盖火灾物理世界理解；其他物理领域只作为未来扩展，不进入当前数据、实验或论文主张。

## 4. 非目标

- 不建设实际消防控制或安全决策系统。
- 不把新的 fire/smoke detector 当作论文主要贡献。
- 不把若干数据集上的割裂简单问答拼接成论文主贡献。
- 不在 v1 扩展到天气、机器人、一般流体或其他非火灾物理领域。
- 不承诺覆盖所有火灾场景、建筑类型或野外火灾域。
- 不在 v1 中实施 `4.升级拓展` 的进阶方向。
- 不在许可证、划分和评测协议冻结前产生论文主结果。

## 5. 成功标准

- 数据：所有计分样本均有许可结论、来源、哈希、单位、case 组和派生链；零已知 split 泄漏。
- 契约：样本、预测和评分均通过机器校验；同一冻结输入可重建相同 benchmark 清单。
- 评测：至少一个朴素基线、一个任务专用基线、一个 LLM 基线和一个工具增强基线；报告 95% 置信区间。
- 科学性：主结论在预注册指标上成立，包含消融、OOD、误差分类和限制说明。
- 研究顺序：先完成 benchmark 与基线，依据预注册失败分析决定是否以及如何提出方法；不得提前固定只为包装贡献的方法。
- 可复现性：每个论文数字可由固定代码、配置、数据清单和原始预测重算。
- 交付：匿名评审包不泄漏身份/私有测试标签，公开包不违反第三方条款。
