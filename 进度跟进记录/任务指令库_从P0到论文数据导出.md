# 任务指令库：从 P0 源码基线到论文数据导出

## 使用方法

通常不需要复制下面的长指令。新窗口发送 `多窗口开发执行手册.md` 中的最短启动指令，代理会从 `CURRENT_STATUS.md` 找到唯一任务，再读取本文件对应任务包。

每个任务的通用要求：先读 `AGENTS.md`、当前状态、最近会话记录和关联设计；先查 Git；只做一个任务 ID；原始数据只读；不读 `4.升级拓展`；未获授权不安装包、不 commit、不 push；结束时更新三件套（状态、会话记录、下一提示词）。

## P0 源码可追溯基线

### 0. P0-REPO-BASELINE-001 首个源码基线

指令：审计当前 UNBORN 仓库的全部未跟踪文件、remote、秘密/身份/大文件和忽略规则，生成初始源码 manifest 与 SHA-256。向用户展示拟提交范围并获得当前窗口的明确授权后，创建本地 initial commit；禁止 push。若用户不授权 commit，则任务保持 `BLOCKED`，不得进入 P1 或生成正式结果。

交付：pre-commit 审计、源码 manifest/hash、首个本地 commit SHA、更新后的 Git/会话基线。

门禁：工作区基线可由 Git 重建；没有秘密、原始数据、私有测试或临时产物入库；remote 未被写入；下一任务 `P1-DATA-001`。

## P1 数据与论文可行性

### 1. P1-DATA-001 数据清单、许可与用途资格

指令：为 D01-D11 生成逐文件相对路径/大小/SHA-256 manifest，核验来源版本、获取日期和官方许可，分别判断训练、开发、测试、派生发布和再分发资格；未知项必须 `BLOCKED`。机器配置保持占位源 `eligible=false`。

交付：版本化 manifest 与自身哈希、许可证据表、更新后的 `configs/data_sources.toml`、数据资产/风险/开放问题。

门禁：所有本地文件可追踪；用途结论有原文证据；未把演示素材转为数据。下一任务只能是 `P1-DATA-002`。

### 2. P1-DATA-002 字段、单位与标准化契约

指令：只针对 P1-DATA-001 允许研究的来源，建立字段字典、shape、时间/坐标/单位、缺失编码和标签格式；未知物理量不猜测。定义 L0->L1 canonical mapping 和可逆转换证据。

交付：逐源字段字典、单位/坐标规范、adapter 设计或最小只读探查脚本、Schema 草案、抽查报告。

门禁：D01-D04 关键字段有来源证据；原单位被保留；转换可重算；下一任务 `P1-DATA-003`。

### 3. P1-DATA-003 数据质量、case 语义与泄漏键

指令：统计缺失、异常、零字节、重复、标签问题和采样特征；确认 case/scenario/family/sequence/near-duplicate/template 分组键，验证 Immersed 配置是否真能形成单变量 pair，不能则降级为配置比较。

交付：质量报告、case/family 注册表、潜在泄漏键、pair 参数 diff、修复/排除策略。

门禁：质量问题均有 disposition；split group 可执行；视觉空标签语义明确或阻塞；下一任务 `P1-RESEARCH-001`。

### 4. P1-RESEARCH-001 会议、相关工作、创新性与污染审计

指令：基于用户/导师给定目标会议或最多两个候选会议，核验最新官方投稿、双盲、数据和 artifact 政策；系统检索相关 benchmark/论文，形成贡献差异矩阵；评估公开数据/题目进入模型预训练和 benchmark 记忆的风险。

交付：venue policy 表、deadline/页数/匿名约束、相关工作矩阵、可证实贡献、污染探针与缓解、更新论文证据矩阵。

门禁：主贡献没有明显重复；会议约束不会推翻数据/发布设计；“首个”等未证实措辞被禁止；下一任务 `P1-FREEZE-001`。

### 5. P1-FREEZE-001 P1 阶段审计与批准

指令：不新增数据研究，逐条审计 P1 总出口；将 D-002/D-003/D-004/D-006 等相关 `PROPOSED` 决策提交用户/导师批准、拒绝或修改。生成 P1 freeze manifest。

交付：P1 审计报告、批准后的决策、冻结 manifest/hash、遗留风险与 P2 输入清单。

门禁：许可、字段、质量、case 语义和论文可行性均达标或明确收缩范围；下一任务 `P2-ONTOLOGY-001`。

## P2 契约、划分与评测冻结

### 6. P2-ONTOLOGY-001 任务与物理机制 ontology

指令：冻结 9 个子任务的标签空间、机制层级、风险状态、缺失/拒答、证据和 physical violation taxonomy；标明每个标签的可观测性与 gold origin。

交付：ontology 文件、标签定义/例子/反例、任务输入输出矩阵、专家待确认项。

门禁：同名标签无歧义；不从视觉数据推断不可观测 CFD 真值；下一任务 `P2-SCHEMA-001`。

### 7. P2-SCHEMA-001 样本/预测 Schema 与语义 validator

指令：把 ontology 落到版本化 sample/prediction Schema，覆盖 T1/T2/T3 条件字段、ID、provenance、trace、评分私有字段；实现正向/负向/边界测试和跨字段语义校验。

交付：Schema、validator、fixtures、迁移/版本规则、测试。

门禁：9 子任务均可机检；私有 gold 不进入公开 prediction；非法单位/ID/证据引用被拒；下一任务 `P2-SPLIT-001`。

### 8. P2-SPLIT-001 group-first 与 OOD 划分

指令：先按 case/family/sequence/重复组划分，再切窗/增强/派生题；构造 ID、火源/通风/HRR/布局 OOD 和 external partitions。比例以 group 可行性决定，不强行套 8:1:1。

交付：split 配置、group 清单、分层统计、混杂报告、稳定 seed/hash。

门禁：group 交集为零；OOD 轴可解释且样本量足够；测试标签隔离；下一任务 `P2-LEAK-001`。

### 9. P2-LEAK-001 匿名 ID 与泄漏审计

指令：实现公开 opaque ID/私有映射，扫描精确/感知重复、视频邻帧、family、模板、文件名、路径和元数据答案线索；对任何泄漏 fail closed。

交付：匿名化工具、私有映射策略、泄漏报告、失败 fixtures、release 扫描规则。

门禁：已知泄漏为零；公开输入不能从名称直接推答案；下一任务 `P2-EVAL-001`。

### 10. P2-EVAL-001 指标、评分器与统计预案

指令：冻结各子任务主/次指标、case/pair 分析单位、失败计分、校准、bootstrap CI、人工评测 rubric 和综合分限制；实现参考评分器原型和可重算测试。

交付：evaluation 配置、指标规范、评分器原型、统计预案、人工 rubric 草案。

门禁：不得以窗口为独立统计单位；无效输出/超时/拒答规则明确；下一任务 `P2-FREEZE-001`。

### 11. P2-FREEZE-001 契约与测试封存

指令：逐条审计 P2 出口；冻结 Schema、ontology、split、泄漏规则、公开/私有边界和评测协议。封存仓库外 test input/gold/private mapping，激活 test embargo 与访问 ledger。还必须在任何 pilot 前冻结 FDS/FD-Gen 的参数空间、seed、case family、预分配 split/OOD 轴、失败处理和生成预算，防止看过 pilot 后适应性设计正式测试。生成版本号与 checksum，不生成模型测试结果。

交付：P2 freeze manifest、test embargo/private manifest、无访问确认、FD-Gen generation plan/hash、批准决策、变更迁移规则、P3 构建输入。

门禁：全部机器测试通过；D-005/D-007 等决策获批；普通开发工作区不可达测试资产；FD-Gen 计划在 pilot 前封存；下一任务 `P3-PIPELINE-001`。

## P3 MVP benchmark

### 12. P3-PIPELINE-001 数据 adapter 与 canonical case pipeline

指令：实现经批准数据源的只读 inventory/adapters/normalizer/case graph，保存父哈希、单位变换、配置、失败记录和确定性输出；不得就地修改原始数据。

交付：核心数据模块、CLI、测试、canonical manifest、质量/失败报告。

门禁：重复构建哈希一致；正常/边界/损坏样本测试通过；下一任务 `P3-BUILD-T1`。

### 13. P3-BUILD-T1 火灾预警样本

指令：实现 T1-A/B/C 的弱信号窗口、异常归因、主动信息选择与 gold 派生；阈值只从 train/calibration 获得并带版本。

交付：T1 builder、样本卡、fixtures、分任务统计、标签重算测试。

门禁：无未来信息；lead time 按 case；非火/扰动/故障可区分；下一任务 `P3-BUILD-T2`。

### 14. P3-BUILD-T2 状态与机制样本

指令：实现 T2-A/B/C 的状态、机制和错误解释一致性任务；证据必须引用 observation ID，错误解释模板 family 与测试隔离。

交付：T2 builder、机制覆盖报告、样本卡、fixtures、测试。

门禁：不是通用 detection 换名；gold origin 清晰；下一任务 `P3-BUILD-T3`。

### 15. P3-BUILD-T3 演化与反事实样本

指令：实现 T3-A/B/C 的趋势、pair 和 trace；自动验证单变量干预，非单变量只能称配置比较；最终答案与中间链分别评分。

交付：T3 builder、pair validator、trace 数据、样本卡、测试。

门禁：无伪反事实；future horizon 无泄漏；下一任务 `P3-SCORER-001`。

### 16. P3-SCORER-001 参考评分器闭环

指令：完成 9 子任务逐样本评分、case/pair 聚合、违规/证据/失败计分和确定性报告；用手算 fixtures 交叉验证。

交付：评分器、CLI、golden tests、metrics schema、版本说明。

门禁：原始预测可重算全部指标；无人工改分；下一任务 `P3-EXPERT-001`。

### 17. P3-EXPERT-001 专家校准与标注协议

指令：在不接触隐藏主测试结果的校准集上训练 rubric，双人独立标注 physical trace/开放解释，计算一致性并裁决；专家不足则收缩任务而非伪称专家标签。

交付：rubric、校准集、盲评记录、一致性/裁决报告、标注版本。

门禁：规定的一致性和专家复核通过，或明确删除不可靠 gold；下一任务 `P3-MVP-RC1-001`。

### 18. P3-MVP-RC1-001 可重复构建 MVP

指令：从冻结输入干净构建两次，生成 `mvp-rc1` public/private 包、数据/benchmark 卡、checksum 和限制；只做 benchmark smoke，不做主模型排名。

交付：MVP release candidate、双构建哈希报告、卡片、P4 输入。

门禁：两次 manifest 相同；公开包无测试 gold/私有 ID；下一任务 `P4-HARNESS-001`。

## P4 Baseline、harness 与 pilot

### 19. P4-HARNESS-001 统一运行系统

指令：实现模型 adapter、prompt/config hash、重试/超时、原始响应、解析、工具轨迹、成本/延迟和不可覆盖 run 目录；公开/私有资产隔离。P4 只允许 train/dev，普通开发进程不得挂载 `FIREWORLDBENCH_PRIVATE_ROOT`。

交付：runner、run manifest、缓存/失败策略、mock adapter、端到端测试。

门禁：失败样本不被删除；指标可从 raw response 重算；下一任务 `P4-BASELINE-NUM`。

### 20. P4-BASELINE-NUM 数值/规则/时序 baseline

指令：实现 chance/majority、领域阈值、传统 ML 与预注册的时序模型；统一输入、调参预算和 case split，只用 train/dev。

交付：baseline 配置/代码、dev pilot run、成本/性能/失败报告。

门禁：不看 test 调参；包含能力下限；下一任务 `P4-BASELINE-VISION`。

### 21. P4-BASELINE-VISION 视觉辅助 baseline

指令：仅在视觉数据获准且任务需要时实现检测/grounding baseline；将 detection 与物理 reasoning 指标分开。若不满足资格，按 DoD 写 `N/A` 决策而不强做。

交付：视觉 baseline 或正式 N/A 决策、dev 结果、域/干扰切片。

门禁：不把视觉准确率包装成 T3 能力；下一任务 `P4-BASELINE-LLM`。

### 22. P4-BASELINE-LLM LLM/multimodal baseline

指令：实现 text-only/table 与合规 multimodal 设置，冻结模型完整 ID、prompt、few-shot、采样、重复、解析和预算；只用 train/dev pilot。

交付：model adapters、prompt registry、dev runs、方差/成本/失败报告。

门禁：模型输入预算透明；无最佳 prompt/test 选择；下一任务 `P4-TOOL-001`。

### 23. P4-TOOL-001 retrieval/plot/tool-use tracks

指令：实现独立 tracks，冻结知识库、绘图、公式/FDS 代理白名单、调用上限和轨迹；通过同模型对照验证是否真的改变答案。

交付：工具 sandbox、轨迹 Schema、dev ablation、成本/失败报告。

门禁：不同信息预算不混榜；工具调用可复放；下一任务 `P4-PILOT-FREEZE-001`。

### 24. P4-PILOT-FREEZE-001 主实验矩阵与预算冻结

指令：仅用 train/dev 汇总 pilot，确定模型、tracks、重复次数、样本量、prompt、预算、失败规则和主/探索矩阵；封存测试访问流程。

交付：pilot 报告、主实验矩阵、预算/时长、冻结配置和 checksum。

门禁：后续不再因 test 表现选模型/阈值；test access ledger 为 `NO_ACCESS_CONFIRMED`；下一任务 `P5-FDGEN-001`。

## P5 Paper-ready 评测

### 25. P5-FDGEN-001 FDS/FD-Gen 受控场景

指令：严格执行 P2-FREEZE 时已经封存的 FDS/FD-Gen 版本、输入、seed、参数空间、family 和 split 预分配，生成单变量 counterfactual/OOD family；记录全部成功与失败 case，不只保留成功样本。不得根据 P4 pilot 表现增加、删除或挑选场景；必要变更必须提升 benchmark 版本并使相关 pilot freeze 失效。

交付：生成输入/输出 manifest、失败日志、pair validator、资源/成本报告。

门禁：场景可重建；与 P2 generation plan diff 为零或有正式版本变更；下一任务 `P5-BENCHMARK-INTEGRATE-001`。

### 25A. P5-BENCHMARK-INTEGRATE-001 正式生成数据重新集成与冻结

指令：在不读取模型 test 结果的条件下，将新生成 cases 完整重走 canonical adapter、sample builders、group/split、匿名化、泄漏审计、gold/trace、Schema 和参考评分器；生成 paper-ready benchmark addendum。不得把新 cases 直接拼进 P3/P4 产物。

交付：canonical/sample manifests、split/leak/gold/scorer 报告、test private manifest、paper benchmark RC 和 checksum。

门禁：所有 P2/P3 不变量重新通过；test embargo 仍有效；benchmark addendum 冻结；下一任务 `P5-FINAL-CALIBRATION-001`。

### 25B. P5-FINAL-CALIBRATION-001 最终 train/dev 适配与封存

指令：仅使用 paper-ready benchmark 的 train/dev，按 P4 冻结的选择协议完成必要训练、阈值拟合和 prompt 适配；不读取逐样本 test input/gold，不改变模型集合或主指标。记录所有选择和访问 ledger。

交付：最终模型/checkpoint/config/prompt hash、train/dev 结果、选择日志、sealed run plan 输入。

门禁：test access 为零；所有正式模型状态冻结；下一任务 `P5-PREREG-001`。

### 26. P5-PREREG-001 正式评测预注册与测试封存

指令：在任何开发者/代理查看 test input、gold 或结果前，冻结假设、主次指标、统计 family、模型矩阵、消融/稳健性、重复次数、排除和停止规则；核验 embargo 与 access ledger，签署配置/hash。

交付：预注册文档、测试访问记录、冻结 run plan、checksum。

门禁：任何后改需新版本并标探索性；下一任务 `P5-MAIN-001`。

### 27. P5-MAIN-001 一次性执行正式主矩阵

指令：严格按冻结计划通过批量 harness 运行 ID/OOD/external/counterfactual 主实验；普通开发者/代理不逐样本浏览 test input，runner 不读 gold，隔离 scorer/custodian 才可读取 gold。保存 raw response、失败、成本、环境和全部访问 ledger；不中途按结果改 prompt/模型。

交付：不可覆盖 run directories、run index、完整性/失败报告。

门禁：所有计划 run 有成功或可解释失败状态；下一任务 `P5-ABLATION-001`。

### 28. P5-ABLATION-001 预注册消融

指令：只执行预注册消融，每次只改变一个声明因素；额外发现标探索性，不混入主假设。

交付：消融 runs、参数 diff、配对结果索引。

门禁：因子隔离成立；下一任务 `P5-ROBUST-001`。

### 29. P5-ROBUST-001 稳健性和压力测试

指令：执行预注册噪声、缺失、传感器故障、视觉退化、措辞变化和工具故障测试；语义/标签不变的变换需验证。

交付：压力测试 runs、变换 manifest、性能/失败/成本切片。

门禁：不以标签变化伪装鲁棒性；下一任务 `P5-STATS-001`。

### 30. P5-STATS-001 冻结统计结果

指令：从 raw predictions 重新评分，在 case/pair 层生成主指标、95% CI、效应量、多重比较和成本/失败；不手工编辑 metrics。

交付：逐样本/逐 case 分数、聚合 JSON/CSV、统计脚本、checksum。

门禁：结果可从 raw outputs 一键重算；下一任务 `P5-ERROR-001`。

### 31. P5-ERROR-001 盲化误差分析

指令：按预注册抽样和 taxonomy，在隐藏模型身份下分析感知/状态/机制/因果/时间/物理/不确定性/工具/格式错误，保留负结果。

交付：误差标签、抽样清单、一致性/裁决、代表案例索引。

门禁：案例不是事后挑选；下一任务 `P5-CLAIMS-FREEZE-001`。

### 32. P5-CLAIMS-FREEZE-001 结果与论文主张冻结

指令：逐项更新论文证据矩阵；主张无证据则降级/删除，确认探索/验证边界，冻结用于论文的 run IDs 和结果 hash。

交付：claims-evidence matrix、结果 freeze manifest、限制/负结果清单。

门禁：没有悬空主张；下一任务 `P6-PAPER-TABLES-001`。

## P6 论文数据生成与导出

### 33. P6-PAPER-TABLES-001 论文表格数据

指令：仅从 P5 冻结结果用脚本生成主表、附录表和每个单元格 provenance；禁止从聊天或手工 Excel 抄数。

交付：机器可读 CSV/JSON、可选 LaTeX 表、table spec、run/metric 映射。

门禁：每个数字可追到 sample scores/raw run；下一任务 `P6-PAPER-FIGURES-001`。

### 34. P6-PAPER-FIGURES-001 论文图数据与渲染

指令：为每张图冻结 source data、绘图 spec、脚本、样式和渲染件；图中汇总与表格使用同一结果源。

交付：figure data CSV/JSON、spec、脚本、PDF/PNG、caption facts。
