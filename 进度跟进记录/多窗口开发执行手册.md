# FireWorldBench 多窗口开发执行手册

本手册用于把长项目拆成多个可替换的对话窗口。聊天记录不是事实源；仓库中的状态、会话记录、配置、manifest、测试和 run 产物才是事实源。

## 1. 最省 token 的日常用法

每次新开窗口只发送下面一句：

```text
请继续当前仓库根 `5.项目实现/v1`。先读 AGENTS.md，并严格按其中“新窗口最小读取集”读取 CURRENT_STATUS.md、其指定的 source_session、NEXT_SESSION_PROMPT.md 和任务库当前小节；验证 handoff 一致后只执行唯一任务。不要复述背景、不要读根 README 或扫描历史。未经我明确允许，不安装/下载，不执行 Git 写操作，不读取仓库外 `../../4.升级拓展`。
```

正常情况下不需要把上一窗口的长回复、PDF 内容或历史命令再次粘贴给新窗口。

## 2. 任务状态机

```text
TODO -> IN_PROGRESS -> DONE
                    -> BLOCKED
                    -> SUPERSEDED
```

- 一个窗口只允许一个主任务 ID 处于 `IN_PROGRESS`。
- 子步骤可以很多，但不得顺带启动下一任务。
- `DONE` 必须满足该任务门禁和 `Definition_of_Done.md`。
- 中断时通常保持 `IN_PROGRESS`；只有确有外部阻塞才用 `BLOCKED`。
- 阶段冻结任务必须通过后，才能进入下一阶段。

## 3. 标准循环

1. 启动：发送上面的最短启动指令。
2. 执行：当前窗口读取本地文件，自行完成唯一任务与验证。
3. 收尾：任务完成时发送“正常收尾指令”；上下文过长时发送“中断交接指令”。
4. 落盘：按“会话记录 -> NEXT_SESSION_PROMPT -> CURRENT_STATUS”的原子顺序写入同一 `handoff_id`；最后做一致性检查。
5. 换窗：新窗口只发送最短启动指令，或粘贴上一窗口生成的续作指令。

具体话术见 `窗口切换与中断恢复模板.md`。

## 4. 何时主动换窗口

满足任一条件就先交接，不要继续开启新的大块工作：

- 一个任务已完成，下一任务属于新的任务 ID。
- 已产生一批重要文件或一次关键冻结，后续工作需要重新审视边界。
- 对话开始重复背景、日志或大段文件内容。
- 下一步需要长时间运行、下载、调用外部 API、训练或大规模评测。
- 当前任务遇到必须由用户/导师决定的选择。
- 用户准备离开，或担心上下文/费用继续增长。

## 5. 节省 token 的硬规则

- 新窗口先读本地文件，不让用户复述背景。
- 聊天只总结关键结论；大表、日志、清单和提示词写入文件并给路径/哈希。
- 数据研究先生成 manifest/统计摘要，后续读取摘要，不反复枚举全部样本。
- 每个正式实验只在聊天报告 run ID、主要结果和异常，完整预测/指标留在 `artifacts/`。
- 不在同一窗口同时做实现、主实验、统计和论文图表。
- 不把未验证的计划写入 `已完成`，否则下一个窗口会在错误事实上继续。

## 6. 全流程顺序

任务定义、前置条件、交付物和门禁全部在 `任务指令库_从P0到论文数据导出.md`。默认严格按其中顺序推进；只有明确标注“可并行”的任务可调整，但每个对话窗口仍只做一个主任务。

阶段冻结点：

- `P0-REPO-BASELINE-001`：首个可重建源码 commit；未授权时阻塞在 P0，绝不 push。
- `P1-FREEZE-001`：数据与论文可行性冻结。
- `P2-FREEZE-001`：Schema、ontology、split、泄漏与评测协议冻结。
- `P3-MVP-RC1-001`：可重复构建的 MVP benchmark 冻结。
- `P4-PILOT-FREEZE-001`：baseline 与主实验矩阵冻结。
- `P5-CLAIMS-FREEZE-001`：正式结果与论文主张冻结。
- `P6-EXPORT-001`：论文机器可读数据包冻结。
- `P7-RELEASE-001`：匿名复现包冻结。

## 7. 不可跨越的边界

- P1 未通过前：不构建正式样本、不训练模型。
- P2 未通过前：不冻结 benchmark、不生成隐藏测试结果。
- P3 未通过前：不宣称 benchmark 已完成。
- P4 pilot 未冻结前：不运行主测试集。
- `P5-PREREG-001` 后：不得根据测试结果改主指标、阈值、prompt 或样本过滤。
- P5 结果未冻结前：不手工制作论文数字。
- P6 导出必须从冻结 run 自动生成，禁止从聊天或手工表格抄数。
