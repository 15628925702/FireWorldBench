# 窗口切换与中断恢复模板

## A. 新窗口最短启动指令

```text
请继续当前仓库根 `5.项目实现/v1`。先读 AGENTS.md，并严格按其中“新窗口最小读取集”读取 CURRENT_STATUS.md、它指定的 source_session、NEXT_SESSION_PROMPT.md 和任务库当前小节；确认三者 handoff_id/READY/current task 一致，不一致就进入恢复模式。只执行唯一任务，不读根 README、不扫描历史。不安装/下载；不要执行 pull/merge/rebase/tag/修改 remote；若本轮任务验收通过，必须完成任务级 commit 并 push 到既有 `origin/main`；不读取仓库外 `../../4.升级拓展`。
```

## B. 正常完成后的收尾指令

```text
当前主任务如果已经满足验收标准，请停止扩展范围并完成正式收尾：运行相称检查，完成任务级 `git add` / `git commit` / `git push origin main`，按 SESSION_HANDOFF_TEMPLATE.md 新建本轮记录，更新 CURRENT_STATUS.md 与 NEXT_SESSION_PROMPT.md，同步相关决策/风险/开放问题/需求追踪。最后只给我：任务状态、关键证据、commit SHA、push 结果、未运行检查及原因、唯一下一任务，以及一段可直接粘贴给新窗口的续作指令。若 push 失败，不得写 DONE。
```

## C. 用户日常使用的中断交接指令

这就是用户以后可以直接说的完整版本：

```text
暂停当前任务。按照 `进度跟进记录/窗口切换与中断恢复模板.md` 给我生成交接指令，我让下一个对话继续做。立即停止新增范围；先处理或明确记录仍在运行的命令，再把实际完成、未完成、最后安全点、工作区改动、验证证据和唯一下一动作落盘到本轮会话记录，更新 CURRENT_STATUS.md 与 NEXT_SESSION_PROMPT.md。未通过验收不得写 DONE。最后给我一段可直接粘贴给下一窗口的续作指令。
```

更短的口令也可以：

```text
按中断模板收尾，生成下一窗口续作指令。
```

当前窗口收到短口令后，必须自动展开为上面的完整要求。

## D. 中断会话记录必须增加的内容

除 `SESSION_HANDOFF_TEMPLATE.md` 原有字段外，中断记录必须写：

- 中断时任务状态：`IN_PROGRESS` 或真实 `BLOCKED`。
- 最后安全点：最后一个已完成且可复现的步骤。
- 部分产物：哪些可复用、哪些不完整/不可信、是否需要删除或重建。
- Git 工作区：已有改动、本轮改动、未跟踪文件；不得把不明改动归给当前窗口。
- 源码恢复证据：clean commit；若没有则保存 staged/unstaged patch hash 和未跟踪文件 manifest/hash。
- 运行状态：命令、PID/session ID、开始时间、输出位置；能安全结束的必须结束后再交接。
- 已运行检查：命令、退出码、结果；未运行原因。
- 精确续作动作：下一窗口第一条命令或第一个文件，不写“继续完善”。

## E. 当前窗口最终应输出的续作指令格式

```text
请继续当前仓库根 `5.项目实现/v1` 的任务 {{TASK_ID}}（当前状态 {{IN_PROGRESS|BLOCKED}}，handoff {{HANDOFF_ID}}）。

先读：
1. AGENTS.md
2. 进度跟进记录/CURRENT_STATUS.md
3. 进度跟进记录/{{LATEST_SESSION_LOG}}
4. 进度跟进记录/NEXT_SESSION_PROMPT.md
5. {{TASK_RELEVANT_DESIGN_OR_CONSTRAINT}}

先验证 CURRENT_STATUS、会话记录、NEXT_SESSION_PROMPT 的 handoff ID/state/task 一致，再执行 git status，保护既有改动。上轮最后安全点：{{LAST_SAFE_POINT}}。
已完成且可复用：{{COMPLETED_OUTPUTS_WITH_PATHS}}。
未完成/不可信产物：{{PARTIAL_OUTPUTS_AND_ACTION}}。
已验证：{{COMMANDS_AND_RESULTS}}。
第一步：{{EXACT_FIRST_ACTION}}。
本轮完成标准：{{REMAINING_ACCEPTANCE}}。

不要重复：{{DO_NOT_REPEAT}}。
不要做：安装包、pull、merge、rebase、tag、修改 remote、读取 4.升级拓展或启动下一任务。
完成或再次中断时，继续按交接模板落盘。
```

## F. 上一窗口意外消失、没有交接记录时

```text
上一个窗口意外中断，没有可信交接。请对当前仓库根 `5.项目实现/v1` 进入只读恢复模式：先检查 CURRENT_STATUS、最近会话记录和 NEXT_SESSION_PROMPT 的 handoff_id/state/task；再读 AGENTS.md，检查 git status、diff/源码快照、文件时间、未跟踪文件与仍在运行的相关进程。对照当前任务验收标准判断哪些产物可验证。不要假定聊天里曾经完成了任何事，不要删除或覆盖不明改动，不要安装/下载，也不要读取仓库外 `../../4.升级拓展`。先生成恢复审计；只有在能证明最后安全点并重新形成三文件一致的 READY handoff 后才继续唯一任务；若随后完成验收，仍必须补做任务级 commit 与 push。
```

## G. 真实阻塞时向用户提问的格式

```text
任务 {{TASK_ID}} 被 {{BLOCKER}} 阻塞。
已尝试并有证据的路径：{{ATTEMPTS}}。
不做决定会影响：{{IMPACT}}。
需要你只决定一件事：{{ONE_EXACT_QUESTION}}。
可选方案及代价：{{OPTIONS_AND_TRADEOFFS}}。
在你决定前，我已把状态和交接写入：{{PATHS}}。
```
