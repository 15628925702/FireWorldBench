# FireWorldBench v1 路线图

路线图使用“门禁通过”而非日期承诺。任何阶段不得以“代码能跑”替代出口证据。

| 阶段 | 目标 | 关键输出 | 出口门禁 |
|---|---|---|---|
| P0 初始化与范围冻结 | 建立事实源、约束、可追踪流程和源码基线 | 初始化文档、仓库骨架、开放问题、local initial commit | 检查通过；源码可重建；remote 未推送 |
| P1 数据与论文可行性 | 确认许可、完整性、单位、case 语义，并前置目标会议/创新性核验 | 数据清单/许可/字段/质量报告、会议政策表、相关工作差异矩阵 | 数据用途明确；占位自动排除；主贡献与 venue 约束可成立 |
| P2 契约与划分冻结 | 把研究概念变成可机检样本和无泄漏 split | JSON Schema、ID/匿名化方案、group/OOD split 清单 | 单元/属性测试通过；重复与泄漏扫描为零 |
| P3 MVP benchmark | 生成 T1/T2/T3 最小闭环 | 构建器、样本卡、参考评分器、人工复核批次 | 可重复构建；专家校准通过；版本标记 `mvp-rc1` |
| P4 baseline 与 harness | 建立能力下限和统一模型接口 | 传统/深度/LLM/tool-use baseline、运行清单 | 所有 run 可追溯；失败与成本完整记录 |
| P5 paper-ready 评测 | 完成 OOD、反事实、消融、稳健性和统计分析 | 冻结测试结果、置信区间、误差分类 | 主指标和统计方案未后改；结论证据充分 |
| P6 论文与内部审稿 | 把代码结果映射到论文声明 | 图表、附录、复现说明、限制与伦理 | 论文证据矩阵无悬空主张；内部复核通过 |
| P7 匿名发布与投稿 | 形成合规、可复现的评审包 | 匿名代码/数据说明/模型输出/校验报告 | 双盲、许可、安全、哈希和从零复现实验通过 |

## 关键依赖链

`源码基线 -> 许可/数据完整性 -> 字段语义 -> 样本契约 -> group split/test embargo -> 标签生成 -> 专家校准 -> baseline -> 预注册 -> 冻结测试 -> 论文主张 -> 论文数据导出`
