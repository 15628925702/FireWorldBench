# 术语表

| 术语 | 项目内定义 |
|---|---|
| benchmark | 数据、任务、划分、输入输出契约、评分器和报告协议的版本化整体，不只是数据文件 |
| case/scenario | 上游一次物理实验或仿真场景；切出的窗口不是独立 case |
| family | 共享 base scenario、仅干预设置不同的一组 case |
| observation window | 截止某时刻可见的有限观测，不包含未来数据 |
| physical trace | 初始状态、主导机制、状态转移、结果、证据和约束组成的 gold/候选链 |
| evidence | 可由稳定 observation ID 指向的公开观测，不是模型自造理由 |
| counterfactual pair | 同一 base case 只改变一个已声明变量的 A/B 对；多变量变化只能称配置比较 |
| ID | 与训练域的生成因素组合相符，但具体 case 不重叠 |
| OOD | 至少一个预注册生成因素在训练域未见，且 group 和混杂检查通过 |
| external validation | 来自不同实验/来源域的验证，不参与训练和阈值拟合 |
| placeholder | 海报、README/论文截图或无可验证原始标签的演示素材，永不计分 |
| eligible | 已通过许可、版本、完整性、字段、质量和伦理门禁的用途特定状态 |
| lead time | 相对预注册确认事件，模型在 case 层首次满足稳定预警条件的提前量 |
| physical violation | 输出违反预注册机理、方向、守恒或结构约束的事件 |
| run | 固定代码、数据、配置、模型和评分器下不可覆盖的一次执行记录 |
