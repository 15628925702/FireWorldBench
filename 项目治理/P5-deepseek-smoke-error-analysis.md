# DeepSeek 小样本低分单独分析

本报告只分析规划阶段的小样本，不把它当作 ICLR 论文结果。

## 观测到的现象

先前 51 个 dev 样本中有 41 个拿到可解析输出，10 个因网络握手/EOF/超时缺失。已解析样本的任务分数为：T1-A 0.167、T1-B 0.333、T1-C 0.000、T2-A 0.500、T2-B 1.000、T2-C 1.000、T3-A 0.333、T3-B 1.000、T3-C 1.000。

## 原因判断

1. **T1-C 的 0 分主要是题面契约问题。** 早期版本把完整观察上下文直接给了模型，却要求模型完成“下一步查询”决策，模型没有真正面对交互式不确定性。当前版本只展示初始观察，并显式给出一个候选查询和 stop 选项；这属于任务接口修复，不能把早期 0 分解释成模型的最终能力。
2. **T2-A 的 0.5 反映规则边界和自然语言状态词不一致。** 同一个温度过程可能被模型称为 forming、developed 或 peak；当前 Gold 已固定阈值和允许标签，但仍需要专家审核确认这些词是否符合论文物理定义。
3. **T1-A、T1-B、T3-A 的低分更像是“数值读表和趋势归纳”错误。** 这些任务需要从多列、跨时间的测量中选出证据，模型容易把某个局部峰值、单位或无关传感器当作主证据。它们是值得保留的模型错误信号，但不能用小样本估计总体性能。
4. **T2-B 的机制判断不能从温度标量单独推出。** 之前的 buoyant_plume 自动标签证据不足，已改为 mechanism_unknown；这会降低表面上“猜对机制”的机会，但提高 Gold 的物理诚实度。
5. **T3-C 的 trace_unknown 是正确的保守行为。** 当前数据没有完整的可审计因果 trace 时，答案应为空 trace/unknown，而不是让模型补写因果链。
6. **T3-B 之前的低分包含 pair 构造问题。** 现在只使用同一 smoke-exhaust 配置、仅改变 ignition longitudinal position 的 D01 文件名对，pair_valid 标志已显式记录；此前结果不能和修复后结果混为一谈。
7. **10 个缺失输出是运行可靠性问题，不是模型能力分数。** 缺失集中在连续 API 请求后，说明需要间隔、重试、超时和逐样本 checkpoint；模型矩阵配置已记录这些运行约束，但本轮不再为验证而消耗预算。

## 结论

当前低分说明三类风险并存：任务接口风险（T1-C）、Gold 语义风险（T2-A/T2-B/T3-C）和真实读表/趋势能力风险（T1-A/T1-B/T3-A）。在扩大实验前，应先用修复后的同一小样本复核接口和 Gold，再把剩余错误按上述类别报告；不能直接宣布“模型不适合”或把低分写成论文结论。
