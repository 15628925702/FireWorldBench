# 05 论文数据导出规范

## 1. 目标

论文数据包必须让每个表格单元格和图形数据点都能追溯到冻结 run、评分器、benchmark/data manifest、配置和代码版本，同时隔离受限数据、隐藏测试标签和双盲身份。

## 2. 目录契约

```text
paper_exports_public/<release_id>/
  README.md
  manifest.json
  checksums.sha256
  provenance/
    run_index.csv
    table_cell_map.csv
    figure_point_map.csv
    text_number_map.csv
    claims_evidence.csv
    versions.json
  tables/
    data/*.csv
    data/*.json
    specs/*.json
    rendered/*.tex
  figures/
    <figure_id>/data.csv
    <figure_id>/spec.json
    <figure_id>/figure.pdf
    <figure_id>/figure.png
  statistics/
    sample_scores_index.csv
    aggregate_metrics.json
    confidence_intervals.csv
    tests_and_effects.csv
    failures_and_costs.csv
  analysis/
    error_taxonomy.csv
    slice_definitions.json
    exclusions.csv
    limitations.md
  environment/
    code_version.json
    data_manifest_refs.json
    resolved_configs/
paper_exports_private/<release_id>/
  README.md
  manifest.json
  checksums.sha256
  private/
    test_label_refs.json
    restricted_run_refs.json
```

`paper_exports_public` 与 `paper_exports_private` 是两个独立根目录，各有自己的 manifest/checksums。公开打包命令只能接受 public 根；任何 public manifest 中出现 private 根、test gold、私有映射或 restricted ref 都必须 fail closed。

## 3. 每个论文数字的最小 provenance

- `paper_item_id`：表号/行/列或图号/series/point。
- `value_unrounded` 与论文显示值、取整规则和单位。
- task/split/slice/metric/statistic。
- run IDs、model/prompt/tool config hash。
- benchmark、data、split、scoring manifest hash。
- clean code commit 与生成脚本版本；若有正式批准的源码归档例外，则记录归档 manifest/SHA-256 和批准决策，不能只写 `dirty=true`。
- 样本或 case/pair 索引；受限时保存私有引用而非公开 ID 映射。

正文数字使用 `text_number_map.csv`，至少记录 manuscript 文件、段落/句子稳定 ID、数字/区间、单位、取整、来源 table/figure/metric/run 和允许的非结果数字类别。样本数、提升幅度、百分比、CI、p 值、成本和失败率都必须映射；年份、模型版本等可进入显式 allowlist。

## 4. 生成规则

- 只读取 `P5-CLAIMS-FREEZE-001` 登记的正式 run。
- 表格、图、正文数字共用同一机器可读结果源。
- 统计与取整在脚本中完成；不得手改 CSV、LaTeX 或绘图点。
- 修复任何数据/评分/导出错误都提升 `release_id`，不覆盖旧包。
- 所有排除、失败、超时、拒答和缺失都进入导出，而非从分母静默删除。
- 公开包不得含原始受限样本、私有映射、隐藏 gold、密钥、个人绝对路径或作者身份。

## 5. `manifest.json` 必填

- release ID、创建时间、目标论文/匿名代号。
- 上游 freeze manifest IDs/hash。
- 文件清单、字节数、SHA-256 和公开/私有分类。manifest 不列出或哈希自身，避免自引用。
- 生成命令、环境、代码版本和 dirty 状态。
- 包含/排除的数据源及许可依据。
- 表/图/主张清单和重建入口。

## 6. 验收

- 在当前记录环境的新输出目录中，仅用导出包机器可读数据重建全部表图，hash 或数值容差一致；跨机器干净环境复现属于 `P7-REPRO-001`。
- 全量 table-cell/figure-point/text-number provenance 无缺失，正文数字扫描没有未映射结果数字。
- 论文显示值与未取整数值、单位、CI、样本数一致。
- public/private、许可证、双盲和秘密扫描通过。
- public/private 两个根的泄漏扫描与各自重建检查通过。
- 每个 `checksums.sha256` 覆盖对应根中除自身外的全部文件，因此会覆盖 `manifest.json`；manifest 不包含自身条目。
