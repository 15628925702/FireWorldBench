# 00 总体架构与实现设计

状态：`SUPERSEDED-AND-REWRITTEN-2026-07-17`。

当前实现架构以 `../../2.方案研究/FireWorldBenchv2(1).pdf` 为唯一核心方案，并由以下文件组成：

- `../docs/ARCHITECTURE_FREEZE.md`：三层九任务、S/I/V、Event/QA、split、指标、QC 与 pilot 冻结。
- `../docs/TASK_PROTOCOL.md`：九任务逐项实施规范。
- `../docs/SOURCE_ROLE_MATRIX.md`：来源准入与报告分域。
- `../schemas/fire_event.schema.json`、`../schemas/qa.schema.json`：机器契约。
- `../src/fireworld/`：新主线实现。

旧 `src/fireworldbench/` 的 T1/T2/T3、physical trace、tool-use/Agent 和 P0-P7 执行链为 `LEGACY-NONCOMPARABLE`。它们保留用于审计，不属于当前架构。

新数据流固定为：

```text
read-only source -> ingest/source record -> Fire Event -> event_group split
                 -> S/I/V task QA -> validation -> model runner -> deterministic scoring
                 -> main / External-CFD / Experiment / Real-Visual OOD reports
```
