## 解释原则

- 核心 PDF 给出“为什么做、做哪三类任务”；本目录规定“怎样生成、怎样运行、怎样判定完成”。
- 本地 MVP 只证明管线可行，不自动支持论文主张。
- paper-ready 版本必须通过许可、数据完整性、FDS/FD-Gen、专家标注、冻结评测和统计门禁。
- 尚未用数据验证的阈值、权重和难度切分都标记为 `TBD_BY_PILOT`，不得提前写死为科学事实。

## 设计变更

任何改变任务、Schema、split、主指标或 paper-ready 范围的修改，必须遵循 `../开发要求约束/变更控制.md`。
