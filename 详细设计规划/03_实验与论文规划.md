# 03 v2 实验与论文规划

## 1. 实验顺序

1. 20-event pilot：L1-2/L2-1/L3-3，至少两轨，chance/rule baseline，成本与标签稳定性。
2. pilot 通过后实现九任务并扩展约 180 个 FDS events、4,000-6,000 QA。
3. 冻结 IID、Geometry/Condition/View-Sensor OOD。
4. 单独接入 External-CFD、Experiment、Real-Image/Video OOD。
5. 统一模型评测；最后才做 FireState Card 轻量 proof-of-concept。

## 2. 主实验报告

- 每个 task/track 的 Accuracy 系主指标和适用补充指标。
- L1/L2/L3 与 Overall 宏平均；外部域单独表格。
- 同时报 event、QA、task、track、source、split 数量。
- 模型失败、拒答、无效 JSON、成本、延迟、ECE/Brier 和 Evidence Accuracy。
- 不用 LLM Judge 生成主榜，不把旧准实验或视觉检测结果包装成 v2 物理推理结果。

## 3. Baselines

pilot 使用 chance/majority 和确定性规则验证闭环。正式模型矩阵目标按核心 PDF 为至少 2 个闭源、4 个开源 MLLM 和 3 类专用模型，但模型、预算和版本必须在执行前冻结。旧 DeepSeek T1/T2/T3 结果不可比较。

## 4. FireState Card

Card 仅由确定性脚本提取关键区域、变量趋势、传感器响应顺序、可见烟气传播方向和当前不确定信息，并与原始输入一并给模型。实验只回答它是否缓解 benchmark 暴露的失败模式；不扩展为 Agent、工具调用、训练框架或四维重建系统。

## 5. 论文证据

所有主张先登记在 `../项目治理/论文证据矩阵.md`。当前阶段只有架构证据，没有 v2 数据规模、性能或改进结论。正式数字必须从冻结预测和 scorer 重算。
