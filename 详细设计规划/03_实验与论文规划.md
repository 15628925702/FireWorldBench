# 03 实验与论文规划

## 1. 论文定位

预期主要贡献不是新 detector，而是：

1. 一个把火灾预警、状态识别和演化推理统一到物理 trace 与证据协议下的 benchmark。
2. 受控 counterfactual、OOD 和解释违规评测，区分表面感知与世界模型式理解。
3. 传统/深度模型与 LLM/multimodal agent/tool-use 的系统比较和失败分析。

该定位需要文献检索和目标会议相关工作分析后才能最终冻结，当前只作为设计假设。

## 2. 预注册研究假设

- H1：通用 LLM/多模态模型在表面状态题上可用，但在早期弱信号、单变量反事实和中间 trace 上显著落后于任务专用模型或物理增强方法。
- H2：显式绘图/结构化表格能提高证据定位，但不必然降低物理违规。
- H3：检索提高术语/机制识别，受控工具使用对 T3 counterfactual 和 trace 的收益更大。
- H4：ID 性能无法可靠预测未见火源位置、通风配置、HRR 或真实实验域的性能。
- H5：答案准确率与物理一致性并非等价；部分模型会“答案对、链条错”。

H1-H5 必须在看冻结测试结果前确认主指标、模型集合和统计比较；否则标为探索性。

## 3. 实验矩阵

| 维度 | 计划设置 |
|---|---|
| 任务 | T1-A/B/C、T2-A/B/C、T3-A/B/C |
| 模态 | sensor、vision、sensor+vision/plot |
| 泛化 | ID、source location OOD、ventilation OOD、HRR OOD、external experiment |
| 系统 | chance/rule、传统 ML、时序 DL、LLM、multimodal、retrieval、plot、tool-use |
| 观测 | 完整、弱信号窗口、缺失、传感器故障/通风扰动 |
| 输出 | 答案、机制、证据、不确定性、trace |

不是所有笛卡尔组合都运行。P3 pilot 用训练/开发集估计可行性、方差和预算，P4 前冻结主矩阵与探索矩阵。

## 4. Baseline 设计

### 4.1 数值/时序

- Majority/chance、持久性和单阈值。
- 逻辑回归/随机森林或梯度树，用统一窗口统计特征。
- CNN-LSTM、TCN、Transformer；相同 split、特征和调参预算。
- T3 数值模型输出还需转为方向/事件，以统一比较。

### 4.2 视觉

- 视觉 baseline 只用于含视觉证据的 T1/T2 子任务。
- bbox 检测、场景分类与物理机制回答分开报告。
- 对负样本、干扰、域外烟羽和低能见度单独切片。

### 4.3 LLM/agent

- text-only：传感器表序列化。
- plot/multimodal：确定性图像化，同一信息内容。
- retrieval：冻结火灾机理知识库，不含测试答案。
- tool-use：白名单绘图、计算、查询和可能的 FDS 代理。
- prompt 只用 train/dev，模板和 few-shot family 与测试隔离。

## 5. 消融

- 去除 physical trace 监督/提示。
- 去除布局、通风、某类传感器或视觉证据。
- 去除 retrieval、plot、工具中的单项。
- 完整观测 vs 主动信息选择。
- 随机 split vs 正确 group/OOD split 仅作为泄漏危害演示，不作为正式性能。
- 答案评分 vs 答案+机制+证据+违规评分。
- simulator-only vs 加入真实实验 external validation。

消融必须一次只改变声明因素；同时变化的设置只能作为系统比较。

## 6. 稳健性与压力测试

- 传感器噪声、漂移、失效、缺失和采样率变化。
- 通风扰动/非火热源/视觉干扰。
- 图像压缩、烟雾遮挡、低照度；只在视觉 track。
- 问题措辞改写、字段顺序和无关证据；保持语义与标签不变。
- 对抗错误解释：热浮力、烟层、通风方向、backlayering、守恒错误。
- 工具故障、延迟和预算耗尽。

## 7. 统计方案

- 分析单位默认是 case/scenario 或 counterfactual pair，不是重叠窗口。
- 报告点估计、95% case-level bootstrap CI 和每个切片的样本/group 数。
- 模型差异使用 paired bootstrap；小样本/离散结果按预注册采用精确或置换方法。
- 多次 API 运行区分样本内随机性和 case 间差异，报告均值、方差和失败率。
- 不以 p 值替代效应量；不过度解读 CI 很宽的小切片。
- 人工评测报告一致性、裁决前后结果和匿名盲评流程。

## 8. 误差分析 taxonomy

- 感知：未读到火/烟/关键传感器变化。
- 定位/证据：结论可能对，但证据引用错或缺失。
- 状态：阶段、火源区、风险级别错误。
- 机制：浮力/顶棚射流/通风/排烟/backlayering 混淆。
- 因果：相关性当因果、反事实方向错、多个变量混杂。
- 时间：响应顺序、事件时刻或未来 horizon 错。
- 物理：守恒、方向、单调性或约束违规。
- 不确定性：过度自信、该拒答时猜测、缺失信息识别失败。
- 工具：选错工具、输入错、读错结果或调用无收益。
- 格式：无效 JSON、字段/单位/证据 ID 不合法。

错误样本抽样规则在查看模型身份前冻结，避免只挑有趣案例。

## 9. 复现与成本

每个正式 run 记录：代码/数据/配置/评分器哈希、依赖锁、硬件、模型/API 快照、prompt、随机种子、原始输出、工具调用、token、费用、延迟和失败。

预算分三级：

- Smoke：每任务少量固定样本，验证接口。
- Pilot：train/dev 子集，估计方差与成本，不看测试。
- Main：冻结矩阵一次性执行，额外运行需登记原因。

## 10. 论文结构草案

1. Introduction：为何 detection/forecast 数值不足以评估火灾世界理解。
2. Related Work：火灾数据/检测/预测、physical reasoning benchmark、LLM agents/tools。
3. FireWorldBench：任务、数据源、physical trace、反事实和 split。
4. Evaluation Protocol：tracks、metrics、统计和人工评测。
5. Baselines and Results：分任务与 ID/OOD，不先给总分。
6. Analysis：工具/模态/trace 消融、物理违规、错误 taxonomy、成本。
7. Limitations/Ethics：仿真到现实 gap、许可、专家标签、安全非部署声明。
8. Reproducibility/Appendix：Schema、prompt、数据卡、完整指标与运行清单。

## 11. 论文主张门禁

以下措辞在证据矩阵未通过前禁止使用：`首个`、`显著优于`、`真实世界有效`、`理解物理`、`专家级`、`通用`。每项主张必须绑定预注册实验、统计结果和限制。

## 12. 投稿前独立审计

- 从干净环境复建一个发布版本和至少一个代表性 baseline。
- 检查 data leakage、许可证、双盲、引用、表格数字和图数据一致性。
- 随机抽取论文数字反向追踪到 `sample_scores.jsonl` 和原始响应。
- 由未参与该模块实现的人复核 split/metric 或至少执行独立脚本审计。
