# 02 任务与评测协议

## 0. 问题定义与两条正交轴

FireWorldBench v1 评测模型能否从不完整、带噪声、多模态的火灾观测中，恢复隐藏物理状态、识别主导机制、预测未来演化，并对受控干预给出物理一致的反事实判断。

- 能力轴：T1 预警与信息需求、T2 状态与机制、T3 演化与反事实。
- 数据轴：仿真、受控实验、传感器时序、视觉和已对齐的多模态组合。
- 数据集不是任务 ontology；数据集只决定可用观测、gold origin、适用切片和外部验证角色。
- 简单 QA 可以作为 smoke、校准或结构化交互形式，但只有映射到统一能力、证据和评分定义后才是正式 benchmark 样本。

完整映射和准入规则见 `06_能力-数据源-任务矩阵.md`。

## 1. 统一输入与输出

输入由 `scenario + public observations + question` 组成。模型输出统一为结构化 JSON，任务可省略不适用字段：

```json
{
  "answer": {},
  "risk_level": "...",
  "dominant_mechanism": "...",
  "causal_chain": ["..."],
  "evidence": ["observation_id"],
  "uncertainty": {"level": "...", "reason": "..."},
  "missing_information": ["..."]
}
```

解析严格遵循预测 Schema。自由文本可保留，但不能代替必需结构字段。

## 2. T1 Fire Early Warning

### T1-A 早期火灾判别

- 输入：3/6/10/20 秒候选窗口的传感器表，可选视觉帧和布局。
- 输出：`fire_forming` 概率/类别、风险等级、预计可确认时间、证据和不确定性。
- 关键对照：真实火灾、非火、通风扰动、传感器故障。
- 主指标：AUPRC；次指标：AUROC、miss rate、false alarm rate、case-level lead time、Brier/ECE、evidence F1、物理违规率。
- 注意：lead time 以预注册确认事件为参考，按 case 计算，不把重叠窗口当独立样本。

### T1-B 异常归因

- 输出类别：`fire`、`non_fire`、`ventilation_disturbance`、`sensor_fault`，允许 `insufficient_information`。
- 主指标：macro-F1；次指标：每类召回、校准、evidence F1、机制违规率。

### T1-C 信息需求选择

- 输入：不完整观测 + 可查询观测集合及成本。
- 输出：下一项观测或停止、理由、预期信息价值。
- 主指标：在预算约束下的决策效用/正确选择率；次指标：达到目标风险判断的查询次数、无效查询率、最终 T1-A 性能。
- 信息价值必须在冻结环境中计算；不允许通过查询名泄漏答案。

## 3. T2 Fire State Recognition

### T2-A 状态识别

- 输出：火灾阶段、火源区域、HRR 档位、风险区域和置信度。
- 主指标：分层标签 macro-F1；次指标：state accuracy、区域距离/IoU（适用时）、校准和不完整字段率。

### T2-B 主导机理识别

- 候选机制：`buoyant_plume`、`ceiling_jet`、`longitudinal_ventilation`、`sidewall_extraction`、`backlayering` 等，经 P2 ontology 冻结。
- 输出可多标签并给证据。
- 主指标：mechanism macro-F1；次指标：evidence precision/recall、层级一致性和置信校准。

### T2-C 物理一致性判断

- 输入：场景证据 + 一段候选解释/模型回答。
- 输出：`consistent/inconsistent/underdetermined`、违规类型、错误机制和证据。
- 主指标：balanced accuracy 或 macro-F1（由校准集类别分布确定）；次指标：violation-type F1、证据 F1。
- 候选错误覆盖浮力、顶棚射流、通风方向、烟层、backlayering、质量/能量约束；生成模板与测试模板 family 隔离。

## 4. T3 Fire Evolution Reasoning

### T3-A 未来趋势预测

- 输入：观测截止点、布局和通风状态。
- 输出：30/60/120 秒温度、组分/烟气、能见度代理和风险区域的方向/区间，可选数值。
- 主指标：trend direction macro accuracy；次指标：event-time error、区间覆盖率、MAE/RMSE、物理违规率。
- 方向和区间优先于不必要的精确数值，避免数值误差掩盖机理能力。

### T3-B 反事实干预推理

- 输入：只改变一个干预变量的 A/B case。
- 输出：哪一 case 风险更高/更低、传感器响应次序变化、backlayering/风险区变化和因果链。
- 主指标：pair ranking accuracy；次指标：counterfactual direction consistency、response-order score、causal edge F1、物理违规率。
- pair 是统计单位，A/B 不拆成独立样本计算显著性。

### T3-C 状态转移追踪

- 输出：`initial_state -> mechanism -> transitions -> outcome` 与证据。
- 主指标：state trace score（节点/边/顺序的预注册组合）；次指标：causal edge F1、transition order accuracy、outcome accuracy。
- 最终答案对但中间链错，不得获得满分。

## 5. 难度与切片

每个样本记录但不泄漏以下切片：

- 模态：sensor、vision、sensor+vision、field/plot。
- 观测完整度：complete、missing-at-random-like、targeted missing、sensor fault。
- 时间：极早、早、中、充分观测；阈值由训练/校准集冻结。
- 物理复杂度：单机制、多机制、竞争机制。
- 泛化：ID、单轴 OOD、external experiment。
- 推理形式：直接、证据选择、反事实、错误解释诊断、工具使用。

难度标签不能由被测模型表现事后定义为主分析标准。

## 6. Track 与公平比较

主报告按 `closed-book`、`retrieval`、`plot`、`tool-use` 分栏。每一栏冻结：

- 可见字段/图像和上下文长度；
- 外部知识库和更新时间；
- 工具白名单、调用上限、超时和成本；
- prompt、few-shot 示例和输出 Schema；
- 重试、温度/采样和解析规则。

不得把不同信息预算的结果直接混成一个排行榜。

## 7. Baseline 最低集合

1. 无技能下限：chance、majority、last-value/简单阈值。
2. 领域规则：温升/组分/响应次序和通风启发式。
3. 传统模型：逻辑回归/树模型（表格）、任务适用的简单视觉检测器。
4. 时序深度模型：CNN-LSTM、TCN、Transformer 至少选择两个，选择依据预注册。
5. LLM：text-only sensor table QA。
6. 多模态：图像/plot + text。
7. 增强：retrieval、plotting、tool-use；各自独立 track。

所有 baseline 都必须输出统一 Schema 或通过版本化 adapter 转换。

baseline 的首要用途是测量能力边界和暴露失败模式。物理 trace prompting、结构化状态记忆、retrieval、仿真工具调用等在 baseline/pilot 之前只登记为候选，不预先作为论文既定方法。

## 8. 评分与综合报告

- 主论文以每个子任务的预注册主指标为核心，不以单一总分替代。
- 可提供 leaderboard summary，但权重必须在测试前冻结，并同时展示全部分项。
- 不同来源先在 case 层聚合，再按预注册宏平均，防止窗口多的数据源支配结果。
- 物理违规率独立报告；不能仅作为解释性文字。
- 统计比较优先用 paired case-level bootstrap；报告差值、95% CI 和实际效应。
- 多模型/多任务检验时报告预注册 family 和适用的多重比较控制。

## 9. 人工评测

- 只用于自动规则难以覆盖的开放解释、trace 合理性和证据充分性。
- 先用不进入测试主统计的校准集训练 rubric。
- 至少两名独立标注者；领域主张需至少一名火灾工程/动力学背景复核者。
- 报告原始一致率和适用的一致性统计（Cohen's kappa/Krippendorff's alpha），再裁决。
- 标注者不看模型身份；输出顺序随机；冲突和裁决原因保留。

## 10. 失败计分

- 无效 JSON：保留原文，结构字段按缺失计，不人工修复。
- 超时/API/工具失败：按预注册最大重试后记失败，不删除样本。
- 内容过滤/拒答：单列并按任务规则计分。
- 证据引用不存在：evidence 得分为 0，并记结构违规。
- 输出超出允许候选或单位：结构/物理违规，不通过模糊字符串匹配纠正。

## 11. 评测冻结物

正式评测前冻结：benchmark manifest、公开/私有 Schema、test group 列表、评分器代码/镜像、主次指标、统计脚本、prompt、模型快照、工具环境、人工 rubric 和随机种子。
