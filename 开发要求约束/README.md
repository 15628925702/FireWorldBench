# 开发要求约束索引

本目录包含 FireWorldBench v1 的强制规则。

- `FireWorldBench_Benchmark_Design_v2.pdf`：研究概念来源，不是可直接执行的工程规格。
- `开发约束总则.md`：项目 MUST / MUST NOT / SHOULD 约束。
- `Definition_of_Done.md`：代码、数据、实验、文档和会话的完成定义。
- `变更控制.md`：范围、数据、Schema、划分、指标和测试集的变更流程。
- `测试集封存与访问控制.md`：P2 后的物理隔离、角色边界、访问 ledger 和污染处理。

发生冲突时按 `AGENTS.md` 的事实源优先级处理；任何例外必须获得用户明确批准并写入决策记录。
