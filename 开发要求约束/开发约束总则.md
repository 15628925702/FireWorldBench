# FireWorldBench v2 开发约束总则

版本：`2.0-draft`
生效：`2026-07-17`

## 权威与范围

- MUST 以 `G:/0-newResearch/2/2.方案研究/FireWorldBenchv2(1).pdf` 为唯一核心方案。
- MUST 固定三层九任务与 S/I/V；MUST NOT 增删任务或把旧 T1/T2/T3、tool-use track、DPO、复杂 FSR/Agent 融入主方案。
- MUST 将旧代码、Schema、配置和结果视为 `LEGACY-NONCOMPARABLE`，除非逐项通过新契约。
- pilot 前 MUST NOT 大规模下载、生成 180 events、训练正式模型或运行主实验。

## 数据与标签

- 原始数据 MUST 只读。所有来源 MUST 先转为 Fire Event，再按 source domain 构建 QA。
- MUST 在切片/增强/QA 前按 `event_group` split；跨 split 泄漏必须为 0。
- 缺失模态 MUST 显式 `null`；不存在的标签 MUST NOT 补造。
- FDS MUST 记录版本、网格、边界、seed、输入、日志、失败与渲染状态。
- 来源、版本、许可、引用、允许用途和再发布范围 MUST 可追溯。
- 同一 event 每任务只生成少量高质量窗口；MUST 同时报告独立 event 和 QA 数量。

## 任务与评分

- 答案 MUST 优先固定标签/有限候选；开放解释只能次要分析。
- MUST 使用冻结的 Accuracy/Component Accuracy/Mean Accuracy 主评分。
- L1/L2/L3 和 Overall MUST 采用任务宏平均；不可比外部域 MUST 单报。
- LLM Judge MUST NOT 进入主排行榜。
- 所有评分 MUST 从 gold、原始预测和版本化配置确定性重算。

## 质量与防捷径

- 公开 ID、文件名、路径、EXIF 和元数据 MUST 不含标签线索。
- 四选一正确位置差异 MUST 不超过 2%；选项长度/风格、窗口时间位置和视觉外观 MUST 审计。
- 困难负例 MUST 视觉相近但物理可区分；模糊、并列、边界样本进入 ambiguous。
- 所有 Event/QA MUST 通过 JSON Schema 和语义 validator。
- FDS 未收敛、数值异常或渲染错误 MUST 不进入正式计分，但失败记录 MUST 保留。

## 工程

- Python `>=3.11`；新主线 MUST 位于 `src/fireworld/`，测试位于 `tests/test_v2_*`。
- 核心逻辑 MUST 有类型注解、确定性 seed、正常/边界/失败测试。
- 危险动作在条件未满足时 MUST fail closed；不得静默捕获异常或伪造空结果。
- 正式 run MUST 不可覆盖并记录代码、配置、数据、模型、prompt、环境和成本。

## Pilot

- MUST 先完成 20 个独立 FDS events、L1-2/L2-1/L3-3、至少两轨、困难负例、自动标签、event split、Schema、简单 baseline 和成本/存储/标签稳定性报告。
- 只有 pilot 门禁全部通过，才能扩展九任务和约 180 events。
- 未经专家或 pilot 确认的阶段、风险、dead-band、tie/same 容差 MUST 保持显式 TBD。
