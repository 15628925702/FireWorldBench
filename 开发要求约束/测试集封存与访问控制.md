# 测试集封存与访问控制

## 1. 目标

从 `P2-FREEZE-001` 起，隐藏测试输入、gold、私有 ID 映射和评分元数据必须与日常开发工作区分离。仅靠“不要看测试集”的提示词不构成防污染证据。

## 2. 物理边界

- 私有测试根目录只能通过 `FIREWORLDBENCH_PRIVATE_ROOT` 提供，必须位于 Git 仓库外。
- 仓库和公开 artifacts 不得包含 test gold、私有映射或私有 scoring metadata。
- test 输入也在预注册前封存；P5 主运行时只由批处理 harness 读取，开发者/代理不逐样本浏览。
- test gold 只允许评分 custodian/隔离 scorer 读取；被测模型、prompt 构建器、普通开发会话永远不可读取。
- aggregate 结果发布前，scorer 只输出预注册指标和必要失败统计，不泄露逐样本 gold。

受保护类别与状态由 `configs/test_embargo.toml` 定义。该配置激活后，任何普通开发对话发现相关路径必须停止读取并登记事件。

## 3. 角色

| 角色 | test input | test gold/private mapping | 允许时间 |
|---|---|---|---|
| 数据/split 构建者 | 只在 P2 封存过程中通过受控脚本处理 | 仅受控构建脚本 | P2 freeze 前 |
| 普通开发者/代理 | 不逐样本读取 | 禁止 | P2 freeze 后 |
| 模型 runner | 批量只读公开输入 | 禁止 | P5-MAIN |
| scoring custodian | 仅评分所需 | 批量只读 | P5-MAIN 后评分 |
| 审计者 | 只看必要索引/聚合 | 需单独授权 | P6/P7 审计 |

角色不能由同一对话自行切换。任何例外必须得到用户明确授权、限定路径/时间/目的，并进入访问 ledger。

## 4. 访问 ledger

所有尝试和授权访问都追加到 `项目治理/测试集访问记录.md`：时间、主体/角色、任务、目的、路径类别、授权依据、实际动作、输出可见范围和是否污染。无访问也在阶段 freeze 时记录 `NO_ACCESS_CONFIRMED`。

## 5. 激活与封存

`P2-FREEZE-001` 必须：

1. 创建仓库外 private root 和公开/私有 manifest。
2. 冻结 test group、opaque ID 和 gold/scoring hash。
3. 将 test 输入和 gold 分区，验证普通工作区不可达。
4. 激活 `test_embargo.toml`，把状态从 `PLANNED` 改为 `ACTIVE`。
5. 写入首次 `NO_ACCESS_CONFIRMED` ledger 条目。

## 6. 污染处理

发现未授权读取、复制、日志泄漏或按 test 个例调参时：

- 立即停止相关任务并登记 `CONTAMINATION_INCIDENT`。
- 隔离受影响 run、prompt、阈值和结论。
- 评估重建测试 split/版本；无法排除污染则废弃该测试版本。
- 不得通过删除 ledger 或只换 sample ID 宣称恢复独立性。
