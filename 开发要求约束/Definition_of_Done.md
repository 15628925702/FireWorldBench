# Definition of Done

只有满足对应类别的全部条件，任务才可标记 `DONE`。不适用项必须写 `N/A` 和理由。

## 通用

- 任务 ID、目标、输入、输出和验收标准明确。
- 实际改动与任务范围一致，无未说明的范围扩张。
- 所有结论都有文件、命令输出或运行产物证据。
- 当前状态、会话记录和下一轮提示词已更新。
- 新风险、决策、开放问题和需求追踪关系已同步。

## 代码

- 正常、边界、失败路径测试齐全；相关测试通过。
- Ruff、mypy 和项目结构检查通过，或有登记且批准的例外。
- 无硬编码个人路径、密钥、测试标签或不可追踪随机数。
- 公共接口、配置项和错误行为已文档化。

## 数据/Schema

- 来源、许可、版本、哈希、单位和父子派生链完整。
- 原始数据未被修改；构建过程幂等。
- Schema 验证、重复/泄漏扫描和 split 不变量测试通过。
- 计分资格明确；占位/演示材料被机器化排除。
- 人工标签已完成规定的校准、一致性和裁决。

## 实验/评测

- 任务、模型、prompt、配置、数据和评分器版本冻结。
- 原始预测、解析失败、工具轨迹、成本、延迟和指标完整保存。
- 统计单位和置信区间符合预注册方案。
- 未根据测试结果做未登记的筛选、调参或指标更换。
- 可由独立命令从原始预测重算最终表格。

## 文档/论文

- 内容与当前代码、配置和数据状态一致。
- 主张已进入论文证据矩阵，且没有用计划结果写成既成事实。
- 引用、数字、图表来源、许可证和双盲信息已检查。
- 限制、负结果和适用边界已写明。
